Introducción a Tailwind — Comparativa con CSS clásico

1) Mismo botón, dos enfoques

2) Padding / Margin — Abreviaturas

CSS: 
padding: 16px; margin: 8px; border-radius: 8px;
Tailwind: 
p-4 (1rem), m-2 (0.5rem), rounded-lg

Regla: p=padding, m=margin, x=eje X, y=eje Y, t=top, b=bottom, l=left, r=right. Ej.: px-6 => padding-left/right 1.5rem; py-2 => padding-top/bottom 0.5rem.

3) Escala de espaciado

p-1 → 0.25rem → 4px

p-2 → 0.5rem → 8px

p-3 → 0.75rem → 12px

p-4 → 1rem → 16px

p-8 → 2rem → 32px

Por defecto, 1rem = 16px. Documentación oficial: Spacing Scale.

4) Valores especiales

Caja centrada (mx-auto)
Margen negativo (-mt-4)

Valor arbitrario: p-[10px] => padding exacto 10px

5) class vs className

HTML puro:

<div class="bg-slate-200 p-2 rounded">Usa class</div>

JSX (React/Next.js):

<div className="bg-slate-200 p-2 rounded">
  Usa className
</div>

En JSX class es palabra reservada, por eso se usa className. Doc React: className.
